# Stellar — 앱 실행 → 별 상세 정보 워크플로우

> 유저가 앱을 실행한 시점부터 별을 탭해 상세 정보를 보는 것까지의 단계별 흐름

---

## STEP 1 — 앱 실행 (S-01 Splash)

| 항목            | 내용                                         |
| --------------- | -------------------------------------------- |
| **동작**        | 로고 표시 + 1.5초 로딩                       |
| **필요 데이터** | 없음                                         |
| **내부 처리**   | 저장된 위치 권한 상태 확인 (`AsyncStorage`)  |
| **분기**        | 권한 있음 → STEP 3 스킵 / 없음 → STEP 2 이동 |

---

## STEP 2 — 위치 권한 요청 (S-02 Permission)

| 항목            | 내용                                                               |
| --------------- | ------------------------------------------------------------------ |
| **동작**        | 권한 요청 다이얼로그 표시                                          |
| **필요 데이터** | 없음                                                               |
| **출처**        | `expo-location` → `requestForegroundPermissionsAsync()`            |
| **결과**        | 허용: 실제 GPS 좌표 확보 / 거부: 기본값(서울 37.5°N, 127.0°E) 사용 |

---

## STEP 3 — 관측자 위치 확정

| 항목            | 내용                                                |
| --------------- | --------------------------------------------------- |
| **동작**        | 화면 전환 전 백그라운드에서 좌표 1회 취득           |
| **필요 데이터** | 위도(lat), 경도(lng)                                |
| **출처**        | `expo-location` → `getCurrentPositionAsync()`       |
| **저장**        | 전역 상태(`zustand`) — 이후 모든 천문 계산의 기준값 |

---

## STEP 4 — 초기 천문 계산

| 항목            | 내용                                                               |
| --------------- | ------------------------------------------------------------------ |
| **동작**        | 별 카탈로그 데이터를 현재 위치·시간 기준으로 지평 좌표로 변환      |
| **필요 데이터** | 위도·경도, 현재 날짜·시각, Hipparcos 카탈로그(JSON 번들)           |
| **출처**        | 위치: STEP 3 결과 / 시각: `new Date()` / 카탈로그: 앱 내 번들 파일 |
| **처리**        | `astronomy-engine` → 적경·적위 → 방위각·고도 변환 (9,096개 별)     |
| **출력**        | 각 별의 `{ x, y, z }` 3D 좌표 배열                                 |

```
적경(RA) / 적위(Dec)          관측자 위도·경도 + 현재 시각
        │                              │
        └──────── astronomy-engine ────┘
                        │
                방위각(Azimuth) / 고도(Altitude)
                        │
                  3D 구면 좌표 (x, y, z)
```

---

## STEP 5 — 3D 성도 렌더링 (S-03 StarMap)

| 항목              | 내용                                                                          |
| ----------------- | ----------------------------------------------------------------------------- |
| **동작**          | Three.js로 천구 씬 초기화 + 별 파티클 일괄 렌더링                             |
| **필요 데이터**   | STEP 4의 좌표 배열, 별 밝기(등급), 별자리 선 연결 정보                        |
| **출처**          | 좌표: STEP 4 결과 / 별자리 선: IAU 88 별자리 연결 데이터(번들)                |
| **Three.js 구성** | `Points` (별 파티클) + `LineSegments` (별자리 선) + `OrbitControls` (회전·줌) |
| **성능**          | 별 9,096개를 단일 `BufferGeometry`로 배치 렌더링 → 60fps 유지                 |

---

## STEP 6 — 사용자 드래그/줌 (S-03 내 인터랙션)

| 항목               | 내용                                              |
| ------------------ | ------------------------------------------------- |
| **동작**           | 한 손가락 드래그로 천구 회전, 두 손가락 핀치로 줌 |
| **필요 데이터**    | 터치 좌표 델타값                                  |
| **출처**           | RN 터치 이벤트 → `OrbitControls` (three.js)       |
| **추가 계산 없음** | 카메라 행렬만 업데이트, 별 좌표 재계산 불필요     |

---

## STEP 7 — 별 탭 (히트 테스트)

| 항목            | 내용                                                                   |
| --------------- | ---------------------------------------------------------------------- |
| **동작**        | 사용자가 별 위치를 탭                                                  |
| **필요 데이터** | 탭 좌표(screen px), 현재 카메라 행렬, 별 3D 좌표 배열                  |
| **처리**        | `THREE.Raycaster`로 탭 지점에서 광선 발사 → 가장 가까운 별 인덱스 반환 |
| **임계값**      | 탭 허용 반경 15px (손가락 오차 보정)                                   |

```
탭 좌표 (screen)
      │
      ▼
Raycaster.setFromCamera()
      │
      ▼
intersectObjects(starPoints)
      │
      ▼
가장 가까운 별 index → 카탈로그에서 상세 데이터 조회
```

---

## STEP 8 — 별 상세 정보 표시 (S-04 Bottom Sheet)

| 항목            | 내용                                                                         |
| --------------- | ---------------------------------------------------------------------------- |
| **동작**        | Bottom Sheet 슬라이드 업, 선택된 별 데이터 표시                              |
| **필요 데이터** | 별 이름(국/영문), 소속 별자리, 겉보기 등급, 거리(광년), 표면 온도, 적경·적위 |
| **출처**        | 모두 Hipparcos 카탈로그(번들 JSON) — 네트워크 요청 없음                      |
| **추가 처리**   | 선택된 별을 3D 뷰에서 하이라이트 (색상·크기 변경)                            |

---

## 전체 흐름 요약

```
앱 실행
  │
  ├─[권한 체크]─────────────────────────────┐
  │                                         │
  ▼                                         ▼
권한 요청 → GPS 취득              기본 위치 사용 (서울)
  │                                         │
  └─────────────┬───────────────────────────┘
                │
                ▼
        위도·경도 + 현재 시각
                │
                ▼
        astronomy-engine
        (적경/적위 → 3D 좌표)
                │
                ▼
        Three.js 렌더링
        (별 파티클 + 별자리 선)
                │
          [사용자 드래그/줌]
                │
            [별 탭]
                │
                ▼
          Raycaster 히트 테스트
                │
                ▼
        Bottom Sheet 표시
        (번들 카탈로그에서 즉시 조회)
```

## 데이터 출처 정리

| 데이터              | 출처                       | 시점              |
| ------------------- | -------------------------- | ----------------- |
| 관측자 위도·경도    | `expo-location` (GPS)      | 앱 시작 1회       |
| 현재 날짜·시각      | `new Date()`               | 렌더링 매 프레임  |
| 별 적경·적위        | Hipparcos BSC5 (번들 JSON) | 앱 번들           |
| 별자리 선 연결 정보 | IAU 88 데이터 (번들 JSON)  | 앱 번들           |
| 방위각·고도 변환    | `astronomy-engine` (계산)  | 위치·시각 변경 시 |
| 별 상세 정보        | Hipparcos BSC5 (번들 JSON) | 탭 시 즉시 조회   |

> **외부 네트워크 요청: 0회** — 모든 데이터가 앱 번들에 포함되어 오프라인에서 완전히 동작합니다.
